2/**
3 * Deliverability posture check (security audit §5c).
4 *
5 * The plugin lets an operator configure three From identities (community
6 * transactional mail, newsletter digests, and the admin_email fallback both
7 * ultimately use) but never checked whether the site can actually send
8 * *legitimately* as them. On default hosting (PHP mail(), no SMTP plugin),
9 * mail from e.g. hello@agnosis.art sent by a shared host fails SPF alignment
10 * and lands in spam — and since Gmail/Yahoo's 2024 bulk-sender rules,
11 * unauthenticated bulk mail is increasingly rejected outright rather than
12 * just filtered.
13 *
14 * This class is a read-only diagnostic surface for Settings → Email Inbox's
15 * "Deliverability" health card — it never changes sending behavior, headers,
16 * or configuration. It reports:
17 *   - whether each configured From identity's domain matches the site's own
18 *     domain (a third-party domain sending on the site's behalf is a much
19 *     more common, and more forgivable, SPF-alignment failure than the
20 *     reverse);
21 *   - whether the From domain publishes an SPF TXT record and a DMARC record
22 *     at all (their *correctness* — do they actually authorize this host —
23 *     is out of scope; a plugin has no way to know a shared host's outbound
24 *     IP without an actual test send, and even that's inconclusive);
25 *   - whether a DKIM record exists under a COMMON selector (dkim_check()) —
26 *     best-effort only; unlike SPF (bare domain) and DMARC (fixed `_dmarc.`
27 *     subdomain), a DKIM record's location depends on a selector the sending
28 *     provider chooses, which this plugin has no way to know for certain.
29 *     "Not found" here means "not found under any of the selectors we
30 *     guessed", never "DKIM isn't configured" — see dkim_check()'s own
31 *     docblock, including the `agnosis_deliverability_dkim_selectors` filter
32 *     for a site that knows its own selector;
33 *   - whether the domain is listed on Spamhaus's Domain Block List
34 *     (dbl_check()) — a real-world confirmed cause of "the email sent fine
35 *     but never arrived", especially for a domain still building sending
36 *     reputation;
37 *   - whether a known SMTP-sending plugin is active, since that's the
38 *     single most common fix for PHP mail()'s poor deliverability.
39 *
40 * DKIM and DBL checks are cached for an hour (see cached()) — DKIM's
41 * selector-guessing issues several sequential DNS queries per identity, and
42 * Spamhaus's DNSBL is a shared public resource with its own fair-use rate
43 * limits; neither should be re-queried on every single admin page load.
44 *
45 * @package Agnosis\Admin
46 */

145        // this short-circuits the real DNS call entirely when filtered to a

368     * `{domain}.dbl.spamhaus.org` for an A record, the standard DNSBL lookup
369     * convention: an NXDOMAIN response (a_lookup() returns 'not_found') means
370     * the domain is clean; any `127.0.1.x`-range response means it's listed,

376     * (rate-limited or malformed query) — treated as 'unavailable' rather
377     * than a real listing, since reporting an error response as "listed"
378     * would be actively misleading.

381     * fair-use query-rate policy; re-querying it on every admin page load
382     * would be both wasteful and a bad way to treat a shared resource other

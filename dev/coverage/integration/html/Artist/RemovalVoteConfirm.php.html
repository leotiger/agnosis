2/**
3 * Email-link vote handler for the community removal-vote flow.
4 *
5 * Artists receive personalised yes/no URLs in the removal-vote notification
6 * email (see DepartureNotification::removal_vote_url()). Those URLs contain a
7 * stateless HMAC token so the artist does not need to be logged in — the same
8 * pattern as VouchConfirm's admission-vote links.
9 *
10 * This class hooks into template_redirect, verifies the token, and — on POST
11 * only — records the vote via Departure::record_vote_on_request(), rendering
12 * a minimal HTML thank-you page or an error page if the token is invalid or
13 * the vote is no longer open. A plain GET (the link as it arrives in the
14 * email) renders a confirm page with a single POST button instead of casting
15 * the vote immediately: corporate mail-security scanners prefetch links to
16 * scan them, and a prefetch alone must never cast a vote (security audit
17 * §2a — the same fix already applied to VouchConfirm/ReviewConfirm/
18 * SubscriptionConfirm).
19 *
20 * This shim did not exist before 0.4.3: DepartureNotification::removal_vote_url()
21 * built links pointing at it, but nothing was ever hooked to consume them —
22 * every removal-vote email link was silently dead (security audit §2e).
23 *
24 * URL shape (all parameters required):
25 *   ?agnosis_removal_vote=1&rid=<request_id>&vid=<voter_id>&vote=<yes|no>&token=<hmac>
26 *
27 * Token construction (must match DepartureNotification::removal_vote_url()):
28 *   hash_hmac( 'sha256', "$voter_id|$request_id|$vote", wp_salt('auth') )
29 *
30 * @package Agnosis\Artist
31 */

3 * Visitor-to-artist contact form.
4 *
5 * Closes a promotion gap flagged 2026-07-12: an artist's biography page
6 * offered no way for a site visitor to actually reach them — only outbound
7 * links (social icons, the portfolio URL) an interested visitor would have
8 * to leave the site to use. This is the inbound counterpart: a popover form
9 * (see Network\SubdomainNavigation's `type=contact` breadcrumb icon and
10 * blocks/contact-form) that lets a visitor write a message in whatever
11 * language they're comfortable with, which this class then:
12 *
13 *   1. Verifies with Cloudflare Turnstile (Core\Turnstile — opt-in, see that
14 *      class), exactly like Admission::apply() and Newsletter\Subscription.
15 *   2. Rate-limits by IP (RateLimiter::check()), once an email address is
16 *      known by that address globally (RateLimiter::check_sender()) — mirrors
17 *      Admission::apply()'s two-tier throttle — and by a THIRD tier scoped to
18 *      (artist, visitor email): RateLimiter::check_sender() again, keyed by
19 *      an action string containing $artist_id, so a visitor can't message the
20 *      same artist more than `agnosis_contact_artist_limit` times (default 2)
21 *      per `agnosis_contact_artist_limit_window_hours` (default 1 hour) —
22 *      configurable in Settings → Email. $artist_id is always the artist's WP
23 *      user ID, the same one regardless of which LinguaForge-translated
24 *      language version of that artist's page the visitor is on, so this
25 *      can't be bypassed by switching languages.
26 *   3. Runs the message through Pipeline::classify_text() against the same
27 *      admin-configured disallowed-content categories EmbedPolicy uses for
28 *      link vetting, plus one contact-form-specific spam/solicitation
29 *      category (see disallowed_categories() below) — "we already dispose of
30 *      filters" per the feature request, reused rather than duplicated.
31 *   4. STORES every submission — sent or rejected — in
32 *      {$wpdb->prefix}agnosis_contact_messages (Core\Activator::create_tables())
33 *      so an admin has a real audit trail via Admin\ContactMessagesPage, not
34 *      just a silent drop. A rejected message is never emailed.
35 *   5. Translates an accepted message into the artist's own language
36 *      (SubmissionTranslator::resolve_artist_lang() +
37 *      SubmissionTranslator::translate_fields(), the same pair
38 *      Notification/ReviewConfirm already use elsewhere) before emailing it.
39 *   6. Emails the artist via CommunityMailer's headers, with `Reply-To:` set
40 *      to the visitor's own address (no such header existed anywhere in this
41 *      codebase before — every prior wp_mail() call only ever set `From`/
42 *      `Bcc`; this is a direct extension of that same "build up the header
43 *      array" convention) so the artist can just hit reply in their own mail
44 *      client — this plugin never learns or stores the artist's real address
45 *      beyond what WordPress core already has in wp_users.
46 *   7. Marks the visitor as having contacted this artist with a short-lived
47 *      `Set-Cookie` (mark_contacted(), same window as the per-artist rate
48 *      limit above) so ContactFormBlock::render_block() can render an inert
49 *      "already contacted" notice instead of the form on the visitor's next
50 *      page load — a simple spam deterrent on top of, not instead of, the
51 *      rate limit itself. Set unconditionally, sent vs. rejected alike, for
52 *      the same "identical response" reason as the REST response itself (see
53 *      below) — the form disappearing is not meant to leak moderation status.
54 *
55 * A rejected message and an accepted one get an IDENTICAL REST response —
56 * see submit()'s final return — deliberately, so the response itself can
57 * never be used as an oracle to probe what the content filter blocks.
58 *
59 * An artist can opt out of this entirely (Artist\NotificationPreferences'
60 * `_agnosis_contact_optout` toggle) — checked before anything else runs
61 * (Turnstile/rate-limit aside), so an opted-out artist's contact icon simply
62 * doesn't render (see SubdomainNavigation) and a direct POST still gets a
63 * clean rejection rather than silently succeeding into a form nobody reads.
64 *
65 * @package Agnosis\Artist
66 */

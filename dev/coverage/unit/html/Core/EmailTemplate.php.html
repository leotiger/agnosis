7 * Artist\CommunityCapNotification, Artist\CommunityBroadcast, and both

9 * card — DOCTYPE, meta tags, header/body/footer table rows, the
10 * `#0d0d12`/`#7c6af7` colour pair, and the CTA button style string —
11 * independently in each `build_*_body()` method. That worked but meant (a)
12 * the same ~20 lines of boilerplate were duplicated a dozen-plus times, and
13 * (b) the header/accent colours were literal string constants, so an
14 * operator who wanted to re-brand outgoing mail (a common ask once a site
15 * has its own visual identity) had no lever anywhere — changing the accent
16 * meant editing PHP in a dozen files by hand.
17 *
18 * This class is that shared shell now: `render()` builds the full HTML
19 * document around a caller-supplied body (and optional extra footer
20 * content), reading its header background and accent colour from Settings →
21 * General → Branding (`agnosis_email_header_bg`/`agnosis_email_accent`,
22 * both new) instead of a hardcoded literal, so every email re-brands
23 * together the moment either setting changes. `button()` is the matching
24 * shared CTA-button renderer, replacing each file's own repeated
25 * `$btn_base` style-string pattern.

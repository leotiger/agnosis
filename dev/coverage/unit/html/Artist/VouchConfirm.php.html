2/**
3 * Email-link vote handler for the artist admission flow.
4 *
5 * Artists receive personalised yes/no URLs in the application notification email
6 * (see AdmissionNotification::vote_url()). Those URLs contain a stateless HMAC
7 * token so the artist does not need to be logged in.
8 *
9 * This class hooks into template_redirect, verifies the token, and — on POST
10 * only — records the vote via Admission::record_vote(), rendering a minimal
11 * HTML thank-you page or an error page if the token is invalid or the
12 * application is no longer pending. A plain GET (the link as it arrives in the
13 * email) renders a confirm page with a single POST button instead of casting
14 * the vote immediately: corporate mail-security scanners prefetch links to
15 * scan them, and a prefetch alone must never cast a vote (security audit §2a).
16 *
17 * URL shape (all parameters required):
18 *   ?agnosis_vouch=1&voter=<user_id>&app=<application_id>&vote=<yes|no>&token=<hmac>
19 *
20 * Token construction (must match AdmissionNotification::vote_url()):
21 *   hash_hmac( 'sha256', "$voter_id|$application_id|$vote", wp_salt('auth') )
22 *
23 * @package Agnosis\Artist
24 */

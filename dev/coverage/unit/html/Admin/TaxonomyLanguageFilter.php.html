3 * Language filter + on-demand medium-translation sync for the Tags/Mediums
4 * wp-admin taxonomy screens (edit-tags.php).
5 *
6 * Every AI-translated term Lingua Forge creates lives in the SAME flat
7 * taxonomy as the admin-curated original (Compat\LinguaForge::sync_taxonomy(),
8 * flagged with TRANSLATED_TERM_META) — on an 18-language site that means the
9 * Tags/Mediums admin screens mix every language's translated copy into one
10 * list (746 tags, 38 mediums observed live), making the screen effectively
11 * unusable for finding or managing the admin-curated vocabulary itself.

13 * This class does three independent things, all applying to BOTH
14 * `agnosis_medium` and `post_tag`:
15 *   1. Scopes the term list to one language at a time — the primary/admin-
16 *      curated vocabulary by default, or any other configured language via a
17 *      new dropdown (LanguageAwareTermsListTable).
18 *   2. Adds a "Sync translations" row action on a primary-language term,
19 *      which calls Compat\LinguaForge::sync_term_across_languages() to
20 *      create any missing translated copy of just that one term on demand
21 *      — see that method's own docblock for why this exists alongside the
22 *      AI-fan-out's own automatic sync.
23 *   3. Adds a one-click "Sync all translations" button next to the language
24 *      dropdown, which calls Compat\LinguaForge::sync_all_terms_across_languages()
25 *      to run the same per-term sync across every primary-language term on
26 *      the current taxonomy in one pass — for filling in a whole backlog of
27 *      missing translations at once, rather than clicking the row action
28 *      one term at a time.

31 */
